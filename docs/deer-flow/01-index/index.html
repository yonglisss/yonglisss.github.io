Technical Guide

01. DeerFlow 教程总览

从跑起来到做实战，系统学习 DeerFlow 2.0 的本地部署、配置、架构和能力扩展。

这套教程解决什么问题

DeerFlow 2.0 不是一个“多 Agent 概念 Demo”。官方给它的定位是 super agent harness：把主 Agent、子 Agent、长期记忆、沙盒、文件系统、工具、技能和消息网关组织到一起，让 AI 能执行较长周期的任务。

如果只看 README，很容易知道它“很强”；但真正上手时，卡点通常不在概念，而在这些问题：

  • 本地到底怎么跑起来？
  • config.yaml 里哪些配置是必须的？
  • 模型、搜索、沙盒、skills 分别影响什么？
  • Docker 和本地开发该选哪个？
  • 跑通以后，怎样改一个能力，而不是只会点网页？

这套教程按“先跑通，再理解，再改造”的顺序写。目标不是复述官方文档，而是给开发者一条可执行的学习路径。

DeerFlow 是什么

DeerFlow 的全称是 Deep Exploration and Efficient Research Flow。

你可以先把它理解成一个能长期执行任务的 Agent 工作台：

  • Lead Agent：负责理解任务、拆解步骤、调度能力。
  • Sub-Agents：把任务拆给多个子 Agent 并行或分工处理。
  • Tools / Skills：把搜索、代码执行、文件读写、外部系统调用封装成可调用能力。
  • Sandbox：把高风险执行放进隔离环境，降低直接污染宿主机的风险。
  • Memory / File System：保存任务状态、上下文和产物，让长任务不只停留在一轮对话里。
  • Gateway / IM Channels：让 DeerFlow 可以通过 Web、Telegram、Slack、飞书、企业微信、钉钉等入口接收任务。

一句话：DeerFlow 想解决的是“AI 不只是回复，而是持续执行”。

适合谁学

更适合这几类人：

  1. 后端 / 全栈工程师
    想理解一个较完整的 Agent 系统怎么组织模型、工具、执行环境和外部入口。

  2. AI 应用开发者
    已经用过 LangChain、LangGraph、CrewAI、AutoGen 等框架，想看一个更工程化的组合方式。

  3. 自动化工作流实践者
    想把研究、数据处理、内容生产、代码修改等任务交给 AI 执行，而不是只让 AI 给建议。

  4. 团队内部工具建设者
    想把 Agent 接进飞书、Slack、企业微信这类工作入口，让它变成团队可用的自动化助手。

如果你只是想找一个开箱即用的聊天机器人，这套教程会偏重；如果你想把 Agent 当成可以改造的工程系统，它会更合适。

学完能做出什么

按这套教程走完，你至少应该能做到：

  • 在本地或 Docker 环境跑起 DeerFlow。
  • 看懂 config.yaml 中模型、搜索、沙盒、channels、skills 的基本结构。
  • 知道一次任务从入口到 Agent 执行的大致链路。
  • 给 DeerFlow 增加或调整一个工具 / skill。
  • 做一个小型实战：例如自动研究一个主题、生成报告、处理文件，或者接入一个 IM 渠道。

这不是“读完就懂所有源码”的路线，而是“先有可运行系统，再逐层打开”的路线。

00. 教程总览

当前这篇。先建立整体地图：DeerFlow 是什么、适合谁、这套教程怎么读。

01. 快速开始：先把 DeerFlow 跑起来

先完成最小闭环：

  • clone 仓库
  • 安装基础依赖
  • 生成配置
  • 配置模型 provider
  • 配置可选搜索能力
  • 运行 make doctor
  • 用 Docker 或本地开发模式启动
  • 跑通第一个任务

02. 配置文件：看懂 config.yaml

重点拆解配置文件：

  • models 怎么配置
  • API key 放 .env 还是直接写配置
  • OpenAI 兼容网关怎么配
  • search provider 怎么选
  • sandbox 配置影响什么
  • DEER_FLOW_PROJECT_ROOT、DEER_FLOW_CONFIG_PATH、DEER_FLOW_HOME 这些环境变量什么时候用

03. 架构主线：一次任务如何被执行

从用户输入开始，追一遍任务链路：入口、Gateway、Lead Agent、工具调用、子 Agent、沙盒、产物返回。

04. 改一个能力：新增或调整 Skill / Tool

不再只使用 DeerFlow，而是动手扩展它：添加一个简单工具，或者把某个内部流程封装成 skill。

05. 实战：做一个自动研究报告 Agent

用一个完整场景串起来：输入主题，搜索资料，整理结构，生成报告，并把结果保存为文件。

建议阅读方式

不要一开始就读源码。

更推荐这样走：

  1. 先按快速开始跑起来。
  2. 遇到配置再回到 config.yaml 那篇查。
  3. 跑通一个任务后，再看架构主线。
  4. 最后才改 tool、skill 或 channels。

Agent 框架最容易陷入“概念先行”。但 DeerFlow 这种工程项目，最好先拿到一个能跑的系统，再逐层拆。这样每个概念都有落点。
