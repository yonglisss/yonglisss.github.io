DeerFlow 教程

  • 00. 教程总览 DeerFlow 教程怎么读：先跑起来，再看地图，按能力模块拆，最后做一个研究型 Agent。
  • 01. DeerFlow 是什么：先知道你在学什么 这篇不装环境，也不读源码。先讲清 DeerFlow 解决什么问题，和普通聊天机器人、RAG、工作流工具有什么区别。
  • 02. 快速开始：先把 DeerFlow 跑起来 用最短路径跑起 DeerFlow：准备环境、安装依赖、配置模型和搜索能力，再确认前后端与一次任务都能正常工作。
  • 03. 配置文件：看懂 config.yaml 把 config.example.yaml 当成地图，先看模型、搜索、MCP、sandbox、tracing 这些会影响运行结果的配置。
  • 04. 整体架构：DeerFlow 到底在调度什么 从 Gateway、Runtime、Lead Agent、middleware、tools、sub-agents、sandbox、MCP、memory、artifacts 看 DeerFlow 的整体地图。
  • 05. 项目结构：从目录看懂 DeerFlow 按官方仓库当前目录看 DeerFlow：前端、后端、Gateway、harness、tools、skills、MCP、sandbox、runtime 各在哪里。
  • 06. 执行链路：一次任务从入口到结果 沿一次 run 追踪 DeerFlow：请求进入 Gateway，runtime 创建任务，Lead Agent 调度工具，最后通过 streaming 和 artifacts 返回结果。
  • 07. Tools 原理：Agent 为什么需要工具 理解 Tool 的边界：模型负责判断和组织，Tool 负责执行动作；好工具要有清晰输入、输出和失败处理。
  • 08. Tools 源码：工具是怎么注册和调用的 从 deerflow/tools 目录看工具注册、内置工具、MCP 工具转换和工具调用的关键文件。
  • 09. Tools 实战：给研究型 Agent 增加一个工具能力 以研究型 Agent 为目标，设计一个低风险工具：输入清楚、输出短、失败可读、能被报告生成流程使用。
  • 10. Skills 原理：把经验沉淀成可复用能力 理解 Skill 的定位：它不是更长的 prompt，而是把稳定工作方法、约束和工具策略沉淀下来。
  • 11. Skills 源码：Skill 是怎么安装、解析和约束工具的 从 installer、parser、validation、permissions、security_scanner、tool_policy 看 Skill 如何进入 DeerFlow。
  • 12. Skills 实战：写 Skill 约束输出质量 为研究型 Agent 写一个 Skill，让它在收集资料、引用证据、生成报告和保存文件时更稳定。
  • 13. Sub-agents 原理：多智能体什么时候值得拆 理解 Sub-agent 的边界：它适合隔离上下文和并行子任务，但不适合把一个模糊任务随便拆碎。
  • 14. Sub-agents 源码：子 Agent 是怎么创建和汇总的 从 subagents/registry、executor、status_contract、token_collector 看子 Agent 的注册、执行、状态和结果汇总。
  • 15. Sub-agents 实战：把研究任务拆给多个 Agent 把一个研究任务拆成边界清楚的子任务：资料收集、源码定位、风险检查，再由主 Agent 汇总。
  • 16. Sandbox 原理：安全执行和文件边界 理解 Sandbox 为什么重要：Agent 一旦能执行命令、读写文件或生成产物，就必须有清晰边界。
  • 17. Sandbox 源码：执行环境和文件路径怎么管理 从 sandbox.py、provider、local_sandbox、middleware、security、tools 看 DeerFlow 如何组织执行环境。
  • 18. MCP 原理：把外部工具接进 Agent 理解 MCP 在 DeerFlow 中的作用：把外部服务包装成 Agent 可发现、可调用、可缓存、可授权的工具。
  • 19. MCP 实战：接一个只读外部服务 用低风险方式接入 MCP：先验证 server、工具列表和权限，再把结果放进研究型 Agent 流程。
  • 20. Memory 原理：长期上下文该记什么 理解 Memory 的边界：它不是聊天记录仓库，而是把稳定、有复用价值的信息注入后续任务。
  • 21. 可观测性：看清一次 Agent 为什么失败 用 tracing、run events、metadata 和日志看清一次 Agent 运行过程，而不是只看最终回复。
  • 22. 排错实战：一次任务为什么失败 把 Agent 失败拆成配置、模型、工具、资料、子 Agent、文件产物几个层级，逐层定位。
  • 23. 终局实战：自己写一个研究型 Agent 把前面能力串起来，设计一个能接收主题、搜索资料、调用工具、生成 Markdown 报告的研究型 Agent。
  • 24. 设计 Agent 任务流 把研究型 Agent 的任务拆成可执行步骤：澄清目标、收集来源、抓取正文、整理证据、生成报告、保存文件。
  • 25. 接入搜索和工具 给研究型 Agent 接入搜索、抓取、整理和文件工具，并明确每个工具在任务流中的位置。
  • 26. 写 Skill 约束输出质量 用 Skill 固定研究型 Agent 的质量标准：来源、证据、不确定性、报告结构和文件验收。
  • 27. 接 MCP / 外部服务 把一个只读外部服务接进研究型 Agent：先验证 server，再接入配置，最后放入报告流程。
  • 28. 生成报告和文件产物 让研究型 Agent 不只返回一段话，而是生成可保存、可检查、可下载的 Markdown 报告文件。
  • 29. 验收与部署 把研究型 Agent 从一次本地演示推进到可长期使用：验收任务、配置安全、日志、回滚和部署边界。

Hermes教程

  • 00. 教程总览 Hermes 源码解读教程怎么读：先会用，再看入口、循环、工具、技能、记忆、Gateway 和 Cron。
  • 01. Hermes 是什么：先知道你在读什么 Hermes 不是普通聊天机器人，而是一个带工具、记忆、技能、Gateway 和定时任务的 AI Agent 运行时。
  • 02. 快速开始：从安装到第一次对话 用最短路径安装 Hermes、完成 setup、选择模型，并跑通第一次 CLI 对话。
  • 03. 配置和模型：config.yaml、.env、provider 怎么配 看懂 Hermes 的配置入口：模型、密钥、工具、Gateway、Memory 和 terminal backend 分别放在哪里。
  • 04. 项目结构：从目录看懂 Hermes 从源码目录理解 Hermes 的入口层、运行层、能力层和后台系统。
  • 05. CLI 入口：hermes 命令怎么启动一次会话 从 hermes 命令、hermes_cli 和 cli.py 看一次终端会话如何启动。
  • 06. Agent 主循环：一次消息如何变成模型调用和工具调用 从 run_agent.py 和 agent/conversation_loop.py 看 Hermes 的核心执行链路。
  • 07. Prompt 与上下文：系统提示词、项目规则、记忆怎么进上下文 看懂 Hermes 如何组装系统提示词、技能索引、项目规则、记忆和运行环境提示。
  • 08. Tools 原理：工具是怎么注册、筛选和执行的 从工具注册、toolset 分组、schema 暴露和 handler 执行理解 Hermes 的工具系统。
  • 09. Tools 源码：registry、toolsets、tool_executor 怎么配合 按源码文件拆解 Hermes 工具从注册、发现、筛选到执行的完整链路。
  • 10. Skills 原理：为什么 Hermes 把经验沉淀成 Skill 理解 Skill 在 Hermes 里的位置：它不是普通提示词，而是可复用、可加载、可维护的过程知识。
  • 11. Skills 源码：Skill 如何被发现、加载和写入 从 skill_utils、prompt_builder 和 skills 工具看 Hermes 的 Skill 生命周期。
  • 12. Memory 与 Session：长期记忆和历史会话怎么参与下一次任务 区分 Memory、User Profile、Session Search 和上下文压缩，理解 Hermes 如何跨会话延续工作。
  • 13. Cron：定时任务怎么保存、触发和投递 从 cron/jobs.py 和 scheduler 看 Hermes 如何管理每日提醒、日报、备份和自动化任务。
  • 14. Gateway：飞书、Telegram、Discord 这类入口怎么接进 Agent 看懂 Hermes Gateway 如何接收外部消息、维护会话、调用 Agent 并把结果投递回平台。
  • 15. MCP：外部工具如何变成 Hermes 工具 理解 Hermes 的 MCP 客户端：stdio、HTTP、SSE 服务器如何接入工具注册表。
  • 16. Delegation：子 Agent 和并行任务怎么跑 理解 Hermes 如何把复杂任务拆给子 Agent，并在隔离上下文里执行。
  • 17. 安全边界：审批、敏感信息、文件路径和工具限制 看懂 Hermes 如何用审批、redaction、上下文扫描、toolset 和路径限制降低 Agent 风险。
  • 18. 调试与排障：日志、doctor、status、常见错误怎么查 给 Hermes 源码阅读和日常使用准备一套实用排障路径。
  • 19. 二开路线：想改一个能力时从哪里下手 把常见二开目标映射到 Hermes 源码目录，避免一上来改错层。
  • 20. 总结：读完后你应该能做什么 收束 Hermes 源码教程，回到入口、主循环、能力模块和二开判断。

OpenClaw教程

  • 00. 教程总览 OpenClaw 源码解读教程怎么读：先会用，再看 Gateway、Agent Runtime、Channels、Tools、Skills、Cron 和插件系统。
  • 01. OpenClaw 是什么：先知道你在学什么 OpenClaw 是本地运行的个人 AI 助手，不只是聊天工具，而是多渠道 Gateway + Agent Runtime + 工具系统。
  • 02. 快速开始：安装、onboard、启动 Gateway 按官方推荐路径安装 OpenClaw，完成 onboard，启动 Gateway，并发起一次 agent 消息。
  • 03. 配置和模型：openclaw.json、状态目录和 auth profiles 看懂 OpenClaw 的配置入口：状态目录、Gateway、Channels、模型、凭证和 sessions 分别放在哪里。
  • 04. 项目结构：从目录看懂 OpenClaw 从源码目录理解 OpenClaw 的 CLI、Gateway、Agent Runtime、Channels、Config、Cron、Plugins 和 MCP。
  • 05. CLI 入口：openclaw 命令怎么启动 从 package.json 和 openclaw.mjs 看 OpenClaw CLI 的启动路径、Node 版本检查和源码/打包模式。
  • 06. Agent Runtime：一次消息如何变成模型和工具调用 从 OpenClaw 的内置 Agent Runtime 看消息、上下文、模型、工具和会话如何串起来。
  • 07. 模型 Provider 与流式响应：OpenClaw 怎么接不同模型 看懂 OpenClaw 的模型层：provider registry、auth profiles、stream implementation 和 failover。
  • 08. Tools 原理：OpenClaw 的工具面是怎么组成的 理解 OpenClaw 工具系统：core、shell、channel、plugin、Tool Search、sandbox 和 policy 如何共同决定可用工具。
  • 09. Tools 源码：agent-tools.ts 怎么组装工具面 按源码拆解 OpenClaw 工具系统：工具创建、参数 schema、policy pipeline、sandbox 和 plugin tools。
  • 10. Gateway：常驻控制平面怎么工作 理解 OpenClaw Gateway 的定位：连接渠道、路由会话、调用 Agent Runtime、投递结果。
  • 11. Channels：多平台消息入口怎么接入 看懂 OpenClaw 的 channel 层：Telegram、Feishu、Slack、Discord 等平台如何变成统一事件。
  • 12. Sessions：会话、历史和状态怎么保存 区分 OpenClaw 的 agent、session、workspace 和状态目录，理解跨渠道连续对话如何保存。
  • 13. Cron：定时任务怎么长期运行 理解 OpenClaw 的 cron 能力：定时触发 agent、保存状态、投递结果和排查失败。
  • 14. Skills：过程知识如何沉淀给 Agent 看懂 OpenClaw 的 skills：它解决的是“遇到某类任务应该怎么做”，不是单纯增加一个工具。
  • 15. Plugins：OpenClaw 的扩展机制 理解 OpenClaw 插件系统：extensions、plugin-sdk、资源包和内置能力如何解耦。
  • 16. MCP：外部工具如何接进 OpenClaw 看懂 OpenClaw 的 MCP 相关目录：agent bundle MCP、配置归一化、外部 server 和工具物化。
  • 17. 安全边界：DM、Sandbox、工具策略和敏感信息 OpenClaw 连接真实消息平台和本地机器，必须理解默认安全策略和风险边界。
  • 18. 调试与排障：从 doctor、gateway status、日志和状态目录开始 给 OpenClaw 使用和源码二开准备一套按层排查的方法。
  • 19. 二开路线：想改 OpenClaw 时从哪里下手 把常见二开目标映射到 OpenClaw 源码目录，避免一上来改错层。
  • 20. 总结：读完后你应该能做什么 收束 OpenClaw 源码教程，回到 CLI、Gateway、Agent Runtime、Channels、Tools 和扩展系统。
