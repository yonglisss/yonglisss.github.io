Technical Guide

26. 写 Skill 约束输出质量

用 Skill 固定研究型 Agent 的质量标准：来源、证据、不确定性、报告结构和文件验收。

为什么终局实战还要 Skill

任务流解决“先做什么后做什么”。

Skill 解决“每一步做到什么标准”。

研究型 Agent 最容易出问题的地方是：看起来写得很完整，但证据不够。

Skill 约束点

建议写进 Skill 的规则：

必须区分事实和判断
关键结论要有来源
拿不到原文时标注不确定
报告结构固定
保存文件后验证路径

这些规则比“写得专业”更有用。

报告质量标准

可以要求报告至少包含：

研究问题
核心结论
证据列表
不确定性
后续可验证动作

如果某个结论没有来源，宁可删掉。

文件验收标准

Skill 里要明确：

生成 Markdown 文件
返回文件路径
确认文件可读取

这能避免 Agent 只在回复里说“已经生成”。

这一篇你要记住

Skill 的作用不是增加文采。

它应该让 Agent 的输出更稳定、更可查、更容易验收。

一个可用的 Skill 片段

## Evidence rules

- 关键结论必须有来源。
- 只看到搜索摘要时，不能写成确定事实。
- 来源冲突时，保留冲突，不要强行合并。
- 生成报告前，列出证据是否足够。

这类规则比“请严谨”有效。

Skill 要和工具配合

如果 Skill 要求来源，但工具没有返回 URL，规则就落不了地。

所以 Skill 和 Tool 要一起设计。

报告 Skill 至少需要工具返回：

标题
URL
正文摘要
抓取状态

迭代方式

先跑三次真实任务，把失败点记录下来，再更新 Skill。

不要凭空写一大堆规则。

练手小作业

把本文的 Evidence rules 放进你的报告 Skill，用一个短主题跑一次；只抽查两条关键结论，反查 URL 和正文证据，并检查文件路径。

完成标准

  • 两条关键结论都能回到来源；
  • 只有搜索摘要的内容没有写成确定事实；
  • 来源冲突或证据不足被保留；
  • 生成的 Markdown 文件可读取。

多走一步（可选）

若有一项失败，只增加或改写一条 Skill 规则，再用同一主题复测。
