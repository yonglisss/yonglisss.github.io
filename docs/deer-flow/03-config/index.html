Technical Guide

03. 配置文件：看懂 config.yaml

理解模型、搜索、tool_search、tool output、sandbox、MCP 和 tracing 配置。

配置是控制面

很多 Agent 行为不是写死在代码里，而是由配置决定：模型、搜索、sandbox、MCP、tool output、channels、tracing 等。

常见配置入口：

config.example.yaml
config.yaml
.env
DEER_FLOW_CONFIG_PATH
DEER_FLOW_EXTENSIONS_CONFIG_PATH

config.example.yaml 是模板，真实密钥不要提交到仓库。

模型配置

排错时先确认：API key、model name、provider endpoint、模型是否支持 tool call，以及 provider 特定参数是否正确。

tool_search

config.example.yaml 提到 tool_search：开启后，MCP tools 不直接全部塞进上下文，而是运行时发现。这能减少上下文占用，尤其适合多个 MCP server 暴露很多工具的场景。

Tool Output Budget

Tool Output Budget Protection 用于防止工具输出过大撑爆上下文。大输出可以持久化到磁盘，模型只看到 preview 和文件引用，需要时再读取完整文件。

Sandbox 配置

模板里能看到：

sandbox:
  use: deerflow.sandbox.local:LocalSandboxProvider
  allow_host_bash: false

官方注释明确：LocalSandboxProvider 不是安全隔离边界，所以 host bash 默认关闭。

MCP 和 Tracing

MCP 配置后要验证 server、OAuth、tools、cache。Tracing 支持 LangSmith 和 Langfuse，长期使用建议尽早开启。

完成标准

你应该知道：配置不是附属品，而是控制面；修改配置后要跑 doctor 或最小任务验证。
