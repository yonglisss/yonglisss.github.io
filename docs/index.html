DeerFlow 教程

  • 00. 教程总览 DeerFlow 教程怎么读：先跑起来，再看地图，按能力模块拆，最后做一个研究型 Agent。
  • 01. 教程总览：从跑起来到写一个研究型 Agent 29 篇完整路线：入门、架构、能力模块闭环，最后完成研究型 Agent。
  • 02. 快速开始：先把 DeerFlow 跑起来 先跑通 DeerFlow，再理解配置、架构和源码。
  • 03. 配置文件：看懂 config.yaml 理解模型、搜索、tool_search、tool output、sandbox、MCP 和 tracing 配置。
  • 04. 整体架构：DeerFlow 到底在调度什么 理解 Gateway、Lead Agent、runtime、middleware、tools、sandbox、MCP 和 streaming。
  • 05. 项目结构：从目录看懂 DeerFlow 按当前官方仓库目录建立源码地图。
  • 06. 执行链路：一次任务从入口到结果 沿着一次 run 追踪 DeerFlow 的真实执行路径。
  • 07. Tools 原理：Agent 为什么需要工具 理解内置工具、sandbox 工具、community tools、MCP tools 和 Skills 的边界。
  • 08. Tools 源码：工具是怎么注册和调用的 从 deerflow/tools 目录理解内置工具、tool_search、task tool、present_file 和 MCP metadata。
  • 09. Tools 实战：给研究型 Agent 增加一个工具能力 以研究型 Agent 为例，设计低风险工具增强流程。
  • 10. Skills 原理：把经验沉淀成可复用能力 理解 Skill 不是 Prompt，也不是 Tool，而是可复用工作方法。
  • 11. Skills 源码：Skill 是怎么安装、解析和约束工具的 理解 installer、parser、validation、permissions、security_scanner 和 tool_policy。
  • 12. Skills 实战：写 Skill 约束输出质量 为研究型 Agent 写一个 research-report Skill。
  • 13. Sub-agents 原理：多智能体什么时候值得拆 理解上下文隔离、专业分工、并行和汇总成本。
  • 14. Sub-agents 源码：子 Agent 是怎么创建和汇总的 理解子 Agent 注册、执行、状态和 token 管理。
  • 15. Sub-agents 实战：把研究任务拆给多个 Agent 设计一个可控的多 Agent 研究流程。
  • 16. Sandbox 原理：安全执行和文件边界 理解 provider、虚拟路径、host bash、文件锁和 artifacts。
  • 17. Sandbox 源码：执行环境和文件路径怎么管理 理解 provider 抽象、local sandbox、security、tools 和文件操作锁。
  • 18. MCP 原理：把外部工具接进 Agent 理解 MCP client、session pool、OAuth、cache 和 tool_search。
  • 19. MCP 实战：接一个只读外部服务 为研究型 Agent 接入低风险只读 MCP。
  • 20. Memory 原理：长期上下文该记什么 理解 Memory 是长期事实，不是聊天记录垃圾桶。
  • 21. 可观测性：看清一次 Agent 为什么失败 用 LangSmith、Langfuse、run events 和 tool output budget 排查任务。
  • 22. 排错实战：一次任务为什么失败 用分层方法定位配置、模型、工具、MCP、sandbox、artifact 和前端问题。
  • 23. 终局实战：自己写一个研究型 Agent 把前面能力串起来，做一个能生成 Markdown 报告的研究型 Agent。
  • 24. 设计 Agent 任务流 把研究型 Agent 拆成解析、计划、搜索、抓取、证据、报告、保存、验证。
  • 25. 接入搜索和工具 为研究型 Agent 接入搜索、抓取、文件读写和 artifact 展示。
  • 26. 写 Skill 约束输出质量 把研究方法写成 Skill，稳定约束报告质量。
  • 27. 接 MCP / 外部服务 把只读 MCP 接入研究型 Agent，扩展资料来源。
  • 28. 生成报告和文件产物 让研究型 Agent 输出可下载、可验证的 Markdown 报告。
  • 29. 验收与部署 为研究型 Agent 制定验收、部署、观测和后续扩展路线。
