这套教程解决什么问题

DeerFlow 2.0 不是简单聊天壳子。官方后端 README 把它描述为 LangGraph-based AI super agent，核心能力包括 Gateway API、Lead Agent、middleware、tools、sub-agents、memory、sandbox、MCP、artifacts 和 streaming。

这套教程按一条完整路径组织：

跑起来 → 看懂配置 → 看懂架构 → 拆能力模块 → 看源码 → 改一个能力 → 写一个研究型 Agent

为什么是 29 篇

19 篇版本能讲清主线，但源码篇不够完整。29 篇版本把关键能力拆成小闭环：

原理 → 源码 → 实战

这样不会只停留在概念，也不会一上来被源码淹没。

内容准确性原则

涉及 DeerFlow 能力和源码路径时，以官方 README、backend/README.md、config.example.yaml 和当前 GitHub 仓库结构为准。拿不到原文时不脑补。

最终你应该能做什么

读完后，你应该能跑起 DeerFlow，理解配置和执行链路，分清 Tool、Skill、Sub-agent、Memory、MCP，定位相关源码，并做出一个能生成 Markdown 报告的研究型 Agent。
