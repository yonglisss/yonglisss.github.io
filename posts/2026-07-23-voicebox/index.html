语音工具往往被拆成几段：一个服务负责克隆声音，一个负责语音转文字，再写脚本把结果接给 AI Agent。Voicebox 想把这条链路装进同一个桌面应用：本地克隆声音、生成语音、全局快捷键听写，还能让支持 MCP 的 Agent 直接开口播报。项目今日进入 GitHub 日榜，单日新增约 565 Star，采用 MIT 许可证。

📌 这个项目是干什么的

Voicebox 是基于 Tauri 的本地优先 AI 语音工作台。它集成 Qwen3-TTS、Chatterbox、Kokoro 等 7 类 TTS 引擎，可从参考音频创建声音档案，也能使用预置声音；官方标注覆盖最多 23 种语言。长文本会按句切分并交叉淡化拼接，Stories 编辑器则可把多个声音排成对话、播客或叙事时间线。

输入侧使用 Whisper：按住全局快捷键说话，转写后可由本地小模型清理口头语和标点，再粘贴到原来的文本框。输出侧提供 REST API 和内置 MCP Server，Claude Code、Cursor、Cline 等客户端可调用 voicebox.speak 播放任务结果。

🔍 为什么值得关注

它的价值不只是“免费语音克隆”，而是把语音输入与输出做成一条可组合的本地工作流。声音样本、录音和模型默认留在设备中；同一声音档案还能绑定描述、后处理效果和 Agent 身份。开发者不必分别维护 STT、TTS、音频文件与 MCP 适配层，做无障碍输入、内容配音或语音通知时，试验成本更低。

项目也给出了明确的工程接口：MCP 服务默认位于本机 127.0.0.1:17493/mcp，并提供转写、声音列表和录音列表等工具。最新版 v0.5.0 重点加入全局听写、本地文本润色与 Agent 语音输出，功能链条已经能从桌面端走通。

🧪 谁适合试，怎么开始

适合经常口述写作的开发者、需要多角色配音的内容团队，以及希望给编码 Agent 增加本地语音提醒的人。macOS 和 Windows 可先从 Release 下载安装包；Linux 暂无预编译包，需要按官方指南从源码构建，Docker 也可运行后端。

安装后建议先用预置声音完成一次短句生成，再导入自己有权使用的参考音频。确认本地服务正常后，可为 Claude Code 添加 MCP：

claude mcp add voicebox --transport http \
  --url http://127.0.0.1:17493/mcp \
  --header "X-Voicebox-Client-Id: claude-code"

⚠️ 使用提醒

“本地运行”不等于低资源占用。不同引擎会下载独立模型，对显存、内存和生成速度的要求差异明显，应先从轻量预置声音和短文本测试。全局自动粘贴在 macOS 还需要辅助功能权限；若把服务从回环地址开放到局域网，应额外检查鉴权与录音数据暴露风险。

更重要的是，声音克隆只应用于本人声音或已获明确授权的素材，并在公开内容中标注合成来源。工具降低了制作门槛，但不会替使用者解决肖像、声音权益和冒充风险。
