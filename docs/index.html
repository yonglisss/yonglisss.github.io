DeerFlow 教程

  • 01. 教程总览：从跑起来到写一个 Agent 重新梳理 DeerFlow 教程路线：先跑通，再按能力模块理解原理、阅读源码，最后自己动手做一个研究型 Agent。
  • 02. 快速开始：先把 DeerFlow 跑起来 用最短路径完成 DeerFlow 2.0 的本地配置、环境检查、Docker / 本地启动和第一个任务验证。
  • 03. 配置文件：看懂 config.yaml 拆解 DeerFlow 2.0 的核心配置：模型、API key、搜索、沙盒、运行目录、skills 和渠道入口。
  • 04. 整体架构：DeerFlow 到底在调度什么 从官方后端架构图出发，理解 DeerFlow 的核心组件：Gateway、Lead Agent、middleware、tools、skills、sub-agents、memory、sandbox 和 artifacts。
  • 05. 项目结构：从目录看懂 DeerFlow 按官方仓库当前目录重新梳理 DeerFlow：前端、Gateway、Agent harness、runtime、sandbox、MCP、skills、memory、uploads 和 tracing 分别在哪里。
  • 06. 执行链路：一次任务从入口到结果 沿着真实运行路径理解 DeerFlow：入口、Gateway、runtime、Lead Agent、middleware、tools、sandbox、artifacts 和 streaming 如何协作。
  • 07. Tools 原理：Agent 为什么需要工具 从 DeerFlow 的工具目录和社区工具生态出发，理解 Tool、MCP、Skill 的边界，以及工具调用为什么必须配合 sandbox 和验证。
  • 08. Skills 原理：把经验沉淀成可复用能力 理解 DeerFlow Skill 的定位：它不是更长的 prompt，也不是 tool，而是可安装、可解析、可校验、会影响 Agent 行为的工作方法。
  • 09. Sub-agents 原理：多智能体什么时候值得拆 理解 DeerFlow 的子 Agent 体系：它解决上下文隔离和任务分工问题，但只有边界清楚、产出可验证时才值得使用。
  • 10. Memory 原理：长期上下文该记什么 理解 DeerFlow 的 Memory：它如何从对话中抽取长期上下文、如何注入系统提示，以及为什么错误记忆比没有记忆更危险。
  • 11. Sandbox 原理：安全执行和文件边界 理解 DeerFlow 的 sandbox：为什么执行必须隔离，LocalSandboxProvider 和 Docker sandbox 的边界在哪里，文件产物如何被安全管理。
  • 12. MCP 原理：把外部工具接进 Agent 理解 DeerFlow 的 MCP 接入：client、session pool、OAuth、tool cache 和 tools 转换如何让外部系统成为 Agent 可调用能力。
  • 13. 可观测性：看清一次 Agent 为什么失败 理解 DeerFlow 的 tracing 与运行观测：LangSmith、Langfuse、metadata、run events、tool output budget 如何帮助定位 Agent 失败原因。
  • 14. 终局实战一：设计一个研究型 Agent 从需求和边界开始，设计一个基于 DeerFlow 的研究型 Agent：输入、输出、工具、Skill、MCP、产物和验收标准。
  • 15. 终局实战二：设计 Agent 任务流 把研究型 Agent 拆成可执行流程：澄清问题、收集来源、抓取正文、整理证据、生成报告、保存文件和返回结果。
  • 16. 终局实战三：给研究型 Agent 写一个 Skill 设计一个 research-report Skill，用稳定流程约束 DeerFlow 研究型 Agent 的资料收集、证据判断、报告生成和文件验证。
  • 17. 终局实战四：接入一个只读 MCP 数据源 给研究型 Agent 增加一个低风险 MCP：先验证 server、配置、工具加载、cache、权限，再把结果并入研究报告。
  • 18. 部署与排错：从本地可用到长期运行 整理 DeerFlow 研究型 Agent 的部署和排错路径：配置、Docker、sandbox、MCP、tracing、artifacts、日志和回滚。
  • 19. 进阶路线：从使用者到改造者 收束 DeerFlow 教程：如何从能跑、会配置、看懂架构，继续走到改 Tools、写 Skills、接 MCP、做自己的 Agent。
