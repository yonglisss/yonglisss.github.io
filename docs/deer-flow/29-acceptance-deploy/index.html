Technical Guide

29. 验收与部署

为研究型 Agent 制定验收、部署、观测和后续扩展路线。

验收标准

最终研究型 Agent 至少要通过：

能解析主题
能搜索来源
能抓正文
能整理证据
能用 Skill 约束报告
能保存 Markdown
能读回验证
能返回 artifact
失败时能说明原因

最小验收任务

研究 DeerFlow 2.0 的 Agent 架构，输出 Markdown 报告。
要求覆盖 Gateway、Lead Agent、Tools、Sandbox、MCP、Memory，并标注来源。

部署前检查

配置路径正确，密钥不进 Git，sandbox 边界明确，MCP 只读工具可用，tracing 可用，artifact 路径稳定，失败日志可查。

后续扩展

更完整 Tools 源码、自定义 Tool 真正落地、Sub-agent 并行研究、MCP OAuth 实战、LangSmith / Langfuse 排错案例、报告自动归档。

完成标准

你有一条从学习 DeerFlow 到改造 DeerFlow，再到做出自己的研究型 Agent 的完整路线。
