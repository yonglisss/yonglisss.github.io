项目愿景

来自官方文档：

“OpenClaw is a personal AI assistant you run on your own devices. It answers you on the channels you already use. The Gateway is just the control plane — the product is the assistant.”

翻译：OpenClaw是在你自己设备上运行的个人AI助手，在你已有的通信渠道上回答你。Gateway只是控制平面——真正的产品是助手本身。

🏗️ 架构设计：为什么这么强大？

三层架构解析

1. Gateway（网关层）

  • 作用：统一控制平面
  • 技术：WebSocket控制平面
  • 特点：单点控制所有会话、渠道、工具和事件
  • 端口：默认 ws://127.0.0.1:18789

2. Channels（渠道层）

支持的25+通信平台：

WhatsApp、Telegram、Slack、Discord、Google Chat、
Signal、iMessage、BlueBubbles、IRC、Microsoft Teams、
Matrix、Feishu、LINE、Mattermost、Nextcloud Talk、
Nostr、Synology Chat、Tlon、Twitch、Zalo、
Zalo Personal、WebChat、macOS、iOS、Android

关键创新：不是让用户适应新工具，而是让AI适应用户的现有工作流。

3. Tools & Skills（工具技能层）

  • 内置工具：浏览器、Canvas、节点、cron、会话管理等
  • 技能系统：可扩展的技能平台（bundled/managed/workspace skills）
  • 安全沙箱：非主会话可在Docker沙箱中运行

🚀 核心功能：远超想象的强大

1. 多设备节点系统

macOS节点：系统命令、通知、Canvas、相机
iOS节点：语音唤醒、Canvas、相机、屏幕录制
Android节点：连接标签、聊天会话、语音标签、Canvas

2. 语音交互革命

  • Voice Wake：macOS/iOS上的唤醒词
  • Talk Mode：Android上的连续语音
  • TTS支持：ElevenLabs + 系统TTS回退
  • 语音唤醒：像真实助手一样随时待命

3. 可视化工作空间

  • Live Canvas：代理驱动的可视化工作空间
  • A2UI：高级UI框架
  • 实时渲染：动态更新的视觉界面
  • 交互控制：直接在Canvas中操作

4. 自动化工作流

  • Cron定时任务：自动化重复工作
  • Webhooks：外部触发集成
  • Gmail Pub/Sub：邮件自动化处理
  • 智能路由：多渠道消息智能分发

🛠️ 快速上手指南

5分钟安装

# 1. 安装Node.js（≥22版本）
# 2. 安装OpenClaw
npm install -g openclaw@latest

# 3. 运行配置向导
openclaw onboard --install-daemon

# 4. 启动网关
openclaw gateway --port 18789 --verbose

连接第一个渠道（以飞书为例）

# 1. 配置飞书机器人
# 2. 在OpenClaw配置中添加飞书渠道
# 3. 测试连接
openclaw message send --channel feishu --message "Hello from OpenClaw"

基础使用命令

# 发送消息
openclaw agent --message "帮我分析这个项目结构"

# 查看状态
openclaw status

# 管理会话
openclaw sessions list

# 配置定时任务
openclaw cron add --name "每日备份" --schedule "0 0 * * *" --command "backup.sh"

🎯 高级功能探索

1. 技能系统开发

# 创建自定义技能
1. 在 ~/.openclaw/workspace/skills/ 创建目录
2. 编写 SKILL.md 文件
3. 实现具体功能
4. 测试并发布

2. 多会话管理

# 同时处理多个项目
会话A：技术开发（Claude模型）
会话B：内容创作（GPT模型）
会话C：数据分析（Gemini模型）

# 会话间通信
openclaw sessions_send --sessionKey "dev-session" --message "代码审查完成"

3. 远程访问配置

# 通过Tailscale远程访问
1. 配置 gateway.tailscale.mode: "serve"
2. 设置安全认证
3. 从任何地方访问你的OpenClaw实例

# 或使用SSH隧道
ssh -L 18789:localhost:18789 user@your-server
