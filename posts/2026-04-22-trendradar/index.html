TrendRadar：比“刷热搜”更有用的开源热点雷达

如果你每天都要盯新闻、看舆情、找选题，TrendRadar 值得放进工具箱。它不是又一个简单的“热榜聚合页”，而是把多平台热榜、RSS 订阅、关键词过滤、AI 筛选、AI 分析和多渠道推送打通成一套可部署的工作流。更重要的是，项目文档足够完整，支持 Docker 与 MCP 接入，已经能让个人或小团队比较低成本地搭一个“自己的热点监控系统”。

📌 这个项目是干什么的

  • 聚合多个热榜平台与 RSS 源，减少来回切站看信息的时间。
  • 支持关键词规则和 AI 兴趣过滤两种模式，适合做行业监测、品牌舆情、选题雷达。
  • 可把结果推送到飞书、钉钉、Telegram、邮件、Slack 等渠道。
  • 还提供 MCP 服务，方便在支持 MCP 的客户端里直接做新闻搜索、聚合、趋势分析和摘要生成。

🔍 为什么值得关注

先记住一个小结：TrendRadar 真正有价值的，不是“抓热点”，而是把“抓到之后怎么筛、怎么看、怎么发”一起做了。

第一，它的上手门槛比很多舆情系统低。README 明确给出 Docker 部署路径，仓库还把 config.yaml、frequency_words.txt、timeline.yaml 等配置职责拆得比较清楚，适合边跑边改。

第二，它不是只做通知。README-MCP-FAQ 里可以看到，项目已经提供搜索新闻、查 RSS、聚合去重、趋势分析、情感分析、生成摘要等工具，更接近“可被 AI 调用的热点数据层”。

第三，它更新节奏还比较活跃。仓库近期仍在迭代，README 展示的版本为 v6.6.1，GitHub 仓库最近一次提交时间也较新，说明不是只靠一波热度冲上来。

🧪 谁适合试，怎么开始

  • 做公众号、短视频、行业研究的人，适合用它盯选题。
  • 做品牌、公关、市场的人，适合拿它做轻量舆情监控。
  • 做 AI Agent、MCP 工具链的人，适合把它当成新闻与趋势数据源。

最短尝试路径可以这样走：先看仓库 README 的 Docker 部署章节，按文档 git clone 后，用 docker compose 跑起来；然后只改三个地方：config/config.yaml、config/frequency_words.txt、docker/.env。如果你本来就在用 Cursor、Cherry Studio 等支持 MCP 的客户端，可以再接上它的 MCP 服务，直接在聊天里查热点和做分析。

⚠️ 使用提醒

  • 它更像“开源工作流组件”，不是开箱即用的商业 SaaS，还是需要自己配环境与推送渠道。
  • README 说明项目会使用外部数据源与 AI 接口，实际稳定性、成本和频率控制要自己评估。
  • 许可证为 GPL-3.0，团队二次开发或嵌入现有产品前，最好先看清合规边界。
  • 如果你只是想“偶尔看看今天热搜”，它可能偏重；但如果你已经有持续监测需求，它的性价比会明显更高。
