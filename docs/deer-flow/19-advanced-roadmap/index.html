19. 进阶路线：从使用者到改造者

给出 DeerFlow 学习和二开路线：会用、会配置、会扩展 Skill / Tool、会接 MCP、会读源码改执行链路。

不要一开始就冲源码

DeerFlow 这类项目，最容易陷入“源码焦虑”：还没跑通，就开始读 Agent、LangGraph、middleware、sandbox。

更好的路线是先建立可运行系统，再逐层深入。

这篇给一条从使用者到改造者的路线。

第一阶段：会用

目标：能把 DeerFlow 跑起来，完成一个简单任务。

你需要掌握：clone 项目、make setup 或 make config、配置一个模型、make doctor、Docker 或本地启动、跑通一个不联网任务、跑通一个研究任务。

这一阶段不要追求理解所有模块。先拿到反馈闭环。

第二阶段：会配置

目标：知道 config.yaml 改什么会影响什么。

重点看：models、API key 和 .env、OpenAI-compatible gateway、search provider、sandbox、skills path、channels、tracing、token budget。

很多问题不是代码问题，而是配置问题。会配置，排错效率会高很多。

第三阶段：会看架构

目标：知道一次任务从入口到结果经过哪些层。

重点理解 Frontend / API / IM Channel、Gateway、Agent Runtime、Lead Agent、Tools、Skills、Sub-agents、Memory、Sandbox、Artifacts。

能画出执行链路，比背目录更重要。

第四阶段：会扩展 Skill

目标：把自己的工作方法沉淀进 DeerFlow。

可以从低风险场景开始：写教程、写报告、排查构建、总结会议、整理资料。

Skill 是最适合普通使用者改造的入口，因为它不一定需要改源码，但能显著改变 Agent 行为。

第五阶段：会扩展 Tool

目标：给 DeerFlow 增加一个确定动作。

适合做成 tool 的能力通常输入输出稳定、经常重复、需要真实执行、prompt 不能可靠完成。

写 tool 时要考虑参数 schema、错误返回、权限边界、日志和验证。

第六阶段：会接 MCP

目标：把外部系统接进来。

从只读 MCP 开始，比如查询、读取、搜索。不要一开始接删除、发布、部署类高风险工具。

掌握 MCP 后，DeerFlow 才能真正进入你的工作流，而不只是独立网页应用。

第七阶段：会读源码和改执行链路

目标：理解 Agent runtime、middleware、subagents、sandbox lifecycle，并能做较深二开。

建议读源码顺序：

  1. config.example.yaml
  2. backend/CLAUDE.md
  3. Gateway 路由
  4. Agent runtime
  5. lead agent
  6. tools / sandbox
  7. skills / memory
  8. subagents
  9. MCP

不要按文件名随机读。

可以做的二开方向

  • 增加团队内部 skills；
  • 接内部知识库 MCP；
  • 加企业 IM 入口；
  • 做任务模板；
  • 做报告生成工作流；
  • 增加安全审计；
  • 优化 token budget；
  • 增加自定义 artifact 展示；
  • 接自己的产品后台。

学习路线建议

如果你是开发者：

跑通 → 配置 → 任务链路 → Skill → Tool → MCP → Sandbox → 源码二开 → 部署治理

如果你只是使用者：

跑通 → 配置模型和搜索 → 写清楚任务 → 检查结果 → 沉淀 Skill

如果你是团队负责人：

跑通 Demo → 选内部场景 → 设计权限 → 接工具 → 做审计和成本控制 → 小范围试用

最后一个判断

DeerFlow 值不值得深入，不取决于它是不是“多 Agent”。

更重要的是：它把模型、工具、沙盒、记忆、skills、sub-agents、Gateway 和 IM 入口放进了一个工程系统里。

如果你只想要聊天，没必要折腾它。
如果你想让 Agent 真正执行任务、生成产物、接入团队工具，那它值得继续拆。

完成标准

读到这里，这套教程的第一版主线就完整了。

你应该已经知道怎么跑起来、怎么看配置、核心概念是什么、目录怎么读、一次任务怎么执行，以及 Tools、Skills、Sub-agents、Memory、Sandbox、MCP 分别解决什么。

后续再深入，就不应该继续泛读，而应该选一个真实任务，把 DeerFlow 改到能稳定完成它。
