Technical Guide

01. 教程总览：从跑起来到写一个 Agent

重新梳理 DeerFlow 教程路线：先跑通，再按能力模块理解原理、阅读源码，最后自己动手做一个研究型 Agent。

这套教程的目标

DeerFlow 2.0 官方把它定义为一个 super agent harness：它不是只做一次问答，而是把 sub-agents、memory、sandbox、skills、tools、MCP 和消息入口组织起来，让 Agent 能处理更长周期的任务。

如果只是泛读 README，很容易停在“它功能很多”。这套教程要解决的是另一个问题：

一个开发者如何从跑起来，走到看懂设计，再走到能改它，最后用它做出一个自己的 Agent。

所以后面的结构会调整为：

入门 → 架构地图 → 能力模块（原理 + 源码 + 小实战）→ 终局实战

这比单独分成“原理篇 / 源码篇 / 实战篇”更顺。因为每个能力都能形成一个小闭环：先知道它解决什么问题，再知道源码在哪里，最后动手改一点。

为什么不只做原理篇和源码篇

原理和源码分得太开，读起来会断。

如果先连续看十篇原理，读者会觉得像概念课；如果直接冲源码，又很容易迷路。更好的方式是按能力模块推进：

Tools：原理 → 源码 → 写一个工具
Skills：原理 → 源码 → 写一个 Skill
Sub-agents：原理 → 源码 → 拆一个任务
MCP：原理 → 接入 → 做一次调用

这样每个模块都能落到一个动作上。

适合谁读

这套教程更适合三类人：

  1. 想部署和使用 DeerFlow 的开发者
    你不一定马上改源码，但至少想知道配置、模型、搜索、sandbox 为什么这样组织。

  2. 想研究 Agent 工程架构的人
    你关心的不只是“多 Agent”概念，而是一次任务怎么进来、怎么调度、怎么调用工具、怎么保存产物。

  3. 想二开 DeerFlow 的工程师
    你想改 tools、skills、MCP、sub-agents，或者把它接进自己的团队工具链。

如果你只是想找一个开箱即用的聊天机器人，这套教程会偏重；如果你想把 Agent 当成工程系统拆开看，它会更合适。

最终要做出什么

这套教程最后会收束到一个完整实战：

自己动手做一个研究型 Agent

它不是重新造一个 DeerFlow，而是在 DeerFlow 之上完成一个可运行的任务流：

  • 接收一个研究主题；
  • 调用搜索或外部工具；
  • 让 Agent 按固定流程收集资料；
  • 用 Skill 约束输出质量；
  • 必要时拆给子 Agent；
  • 生成 Markdown 报告；
  • 保存文件产物；
  • 用日志或 tracing 排查问题。

这个终局项目可以把前面所有模块串起来。

03. 能力模块：每个能力都走一个小闭环

后面会按能力模块组织，每个模块尽量保持这个节奏：

原理：这个能力解决什么问题
源码：实现入口在哪里
实战：改一点，跑一次

第一批模块建议是：

Tools
Skills
Sub-agents
Sandbox
MCP
Memory
Observability

这些是 DeerFlow 作为 Agent harness 的核心能力。

04. 终局实战：自己写一个研究型 Agent

最后几篇不再零散练习，而是围绕一个完整项目：

设计任务流 → 配置工具 → 写 Skill → 接外部工具 → 生成报告 → 验收部署

这比单独写很多“实践技巧”更有目标感。

建议阅读方式

建议按这个顺序读：

  1. 先跑通快速开始。
  2. 遇到模型、搜索、sandbox 问题，再回头看配置。
  3. 跑通一次任务后，看整体架构和执行链路。
  4. 之后按能力模块读：原理、源码、小实战一起看。
  5. 最后做研究型 Agent 大实战。

不要一开始就读所有源码。先有运行反馈，再看源码，理解会快很多。
