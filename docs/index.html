技术教程

这里放更像 GitBook 的教程、手册和项目文档。和主站文章不同，这里优先服务“查阅、照做、复用”。

DeerFlow 教程

  • 01. 教程总览：从跑起来到写一个 Agent 重新梳理 DeerFlow 教程路线：先跑通，再按能力模块理解原理、阅读源码，最后自己动手做一个研究型 Agent。
  • 02. 快速开始：先把 DeerFlow 跑起来 用最短路径完成 DeerFlow 2.0 的本地配置、环境检查、Docker / 本地启动和第一个任务验证。
  • 03. 配置文件：看懂 config.yaml 拆解 DeerFlow 2.0 的核心配置：模型、API key、搜索、沙盒、运行目录、skills 和渠道入口。
  • 04. 整体架构：DeerFlow 到底在调度什么 从官方后端架构图出发，理解 DeerFlow 的核心组件：Gateway、Lead Agent、middleware、tools、skills、sub-agents、memory、sandbox 和 artifacts。
  • 05. 项目结构：从目录看懂 DeerFlow 按官方仓库当前目录重新梳理 DeerFlow：前端、Gateway、Agent harness、runtime、sandbox、MCP、skills、memory、uploads 和 tracing 分别在哪里。
  • 06. 执行链路：一次任务从入口到结果 沿着真实运行路径理解 DeerFlow：入口、Gateway、runtime、Lead Agent、middleware、tools、sandbox、artifacts 和 streaming 如何协作。
  • 07. Tools 原理：Agent 为什么需要工具 从 DeerFlow 的工具目录和社区工具生态出发，理解 Tool、MCP、Skill 的边界，以及工具调用为什么必须配合 sandbox 和验证。
  • 08. Skills 原理：把经验沉淀成可复用能力 理解 DeerFlow Skill 的定位：它不是更长的 prompt，也不是 tool，而是可安装、可解析、可校验、会影响 Agent 行为的工作方法。
  • 09. Sub-agents：多智能体是怎么分工的 理解子 Agent 的作用、适用场景、上下文隔离、结果汇总和常见失控点。
  • 10. Memory：长期记忆如何参与任务 理解长期记忆：哪些信息值得记、哪些不能记、如何避免污染，以及它和上下文、skill 的区别。
  • 11. Sandbox：安全执行和文件系统 理解 sandbox 和文件系统边界：为什么需要隔离、能隔离什么、不能隔离什么，以及本地和 Docker 模式取舍。
  • 12. MCP：接入外部工具和服务 理解 MCP 的作用：什么时候用 MCP、如何配置、如何验证工具可见，以及常见连接和权限问题。
  • 13. 可观测性：LangSmith / Langfuse 追踪 讲清 Agent 为什么需要 tracing，如何观察模型调用、工具链路、成本、延迟和失败原因。
  • 14. 实战一：用 DeerFlow 做一次深度研究 用一个研究任务串起模型、搜索、抓取、来源检查和报告生成，理解 DeerFlow 如何完成资料型任务。
  • 15. 实战二：用 DeerFlow 生成一份技术报告 用 DeerFlow 把一个技术主题整理成 Markdown 报告，重点控制结构、事实来源、风格和最终落盘。
  • 16. 实战三：给 DeerFlow 增加一个自定义 Skill 用一个真实场景演示如何定义、编写、挂载和验证 DeerFlow 自定义 Skill。
  • 17. 实战四：接入一个 MCP Server 用最小路径演示 MCP Server 接入思路：选择服务、配置连接、验证工具、执行任务和排错。
  • 18. 部署：从本地跑到可长期使用 从资源、Docker、配置、密钥、日志、更新和备份角度梳理 DeerFlow 长期运行部署注意事项。
  • 19. 进阶路线：从使用者到改造者 给出 DeerFlow 学习和二开路线：会用、会配置、会扩展 Skill / Tool、会接 MCP、会读源码改执行链路。
